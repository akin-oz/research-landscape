# Email template catalog

Choose the narrowest template that matches the actual purpose. Each copy becomes one `{{relationship_id}}-EMAIL-{{sequence_3}}` record and links to a contact-log entry. Templates do not imply that a message was sent, received, welcomed, or agreed.

| Email purpose | Template | Use only when |
| --- | --- | --- |
| Initial introduction | [initial-contact.md](initial-contact.md) | Beginning a research-focused relationship. |
| Bounded technical guidance | [research-advice-request.md](research-advice-request.md) | Asking one prepared, evidence-grounded question. |
| Discussing a paper | [paper-discussion.md](paper-discussion.md) | A specific paper/method merits a focused exchange. |
| Requesting a meeting | [meeting-request.md](meeting-request.md) | There is a clear discussion purpose and a low-pressure request. |
| Replying to a concrete positive response | [reply-to-positive-response.md](reply-to-positive-response.md) | The prior message directly offered a next step. |
| Post-meeting confirmation and thanks | [post-meeting-thank-you.md](post-meeting-thank-you.md) | A meeting actually took place. |
| Sharing application progress | [application-update.md](application-update.md) | A material, source-supported application milestone exists. |
| One gentle follow-up | [gentle-follow-up.md](gentle-follow-up.md) | A documented waiting period has passed and one reminder is appropriate. |

Keep claims in the email backed by the facts table in that file. Put an open question or inference in the “assumptions to exclude” section until it is verified.
