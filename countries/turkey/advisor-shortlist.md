## Message discipline

Lead with one specific paper/repository/project and one concrete software contribution: workflow automation, provenance, test/CI infrastructure, high-throughput job orchestration, benchmark data, or a reusable Python package. Ask whether an MSc place, a suitable current project, and software-as-a-first-class-thesis-output are feasible. Never assume an advisor uses ASE, pymatgen, a particular cluster, or a public-code workflow unless their source explicitly establishes it.

## Attachments worth preparing

- One-page research statement describing a software-owned, materials-grounded thesis.
- Small public GitHub portfolio with tests, documentation, license, and reproducible environment.
- A brief Materials Atlas/Research Landscape architecture note focused on data provenance rather than product branding.
