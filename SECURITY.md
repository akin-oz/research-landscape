# Security Policy

## Supported versions

Only the default branch is supported while the project is pre-release.

## Reporting a vulnerability

Do not file public issues for suspected security vulnerabilities in future automation, credentials, or data-processing pipelines. Use the repository host's private security-advisory feature when enabled; otherwise contact the maintainers privately through the repository owner. Include reproduction steps, impact, and a safe proof of concept. We will acknowledge reports and coordinate disclosure before publishing details.
