# Entities

`entities/` is the architecture boundary for the repository's first-class knowledge entities. It is intentionally empty of migrated records: future work will add Markdown entity records incrementally without moving or renaming existing material in this phase.

Each child directory is a canonical home for one entity type. Views and country-oriented navigation will reference these records rather than duplicate them. `departments/` is included alongside the requested directories because Department is a first-class entity in the model.
