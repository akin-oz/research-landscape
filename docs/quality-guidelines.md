# Quality guidelines

Write concise, neutral, accessible Markdown. State facts with citations; mark interpretation as interpretation and explain its basis. Use absolute dates, defined time windows, and tables for evidence. Avoid superlatives, prestige proxies, speculation, hearsay, and institutional stereotypes. Use `not available` for missing data, include confidence for material claims, and retain enough calculation detail for independent replication.

For canonical entities, put reusable public facts and their `SRC-*` evidence in
the entity record once; use stable IDs and typed relationships rather than
copied names or inverse edges. Describe what a source does **not** establish.
For reports, disclose scope, evidence window, conflicts, scoring model, and
coverage. For generated files, never edit prose directly: update inputs and run
the documented automation. See [entity authoring](entity-authoring.md) and the
[review process](review-process.md).
