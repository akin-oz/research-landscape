# Adem Tekin — research, group, and software profile

**Interpretation — high confidence:** Tekin’s research has a coherent computational spine: global optimization and high-accuracy intermolecular modelling developed into crystal-structure prediction (CSP), force fields, HPC-enabled materials design, and then data/ML-assisted materials discovery. That path is especially relevant to a candidate who wants to build trustworthy research software around materials rather than only run established packages.

### Patterns extracted from the corpus

1. **Search + validate is recurrent.** CSP/representation generates a tractable space; ML potentials or classical force fields rank it; DFT/AIMD/convex-hull analysis tests important candidates. The borohydride and PNcsp+ studies make this handoff explicit. [borohydrides](https://research.itu.edu.tr/en/publications/hydrogen-storage-in-trimetallic-borohydrides-a-crystal-structure-/) · [PNcsp+](https://pmc.ncbi.nlm.nih.gov/articles/PMC13085239/)
2. **The method is comparatively interpretable.** The PN representation is a transparent chemical-similarity prior, unlike a black-box generative model; PNcsp+ still uses GNN potentials for ranking rather than claiming the prior replaces physics. [PNcsp+](https://pmc.ncbi.nlm.nih.gov/articles/PMC13085239/)
3. **Python is operational, but not the only language.** Public code is Python; course evidence includes Linux, shell, Make, Fortran/C and object-oriented programming. [PNcsp+](https://github.com/tccdem/PNcsp_Plus) · [HBM 801](https://ninova.itu.edu.tr/en/courses/institute-of-informatics/13508/hbm-801/form)
4. **ML is used as a screening/acceleration layer.** The 2025 perovskite study uses gradient boosting, cross-validation, open Python packages (Pymatgen/Matminer/scikit-learn), then DFT validation. That is a sounder conversation entry point than proposing an unscoped “AI materials” project. [PCCP full text](https://pubs.rsc.org/en/content/articlepdf/2025/cp/d4cp04218b)
