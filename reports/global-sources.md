## Evidence protocol for new claims

1. Prefer the current institutional/group page for role, team membership and openings.
2. Prefer a first-party repository, release record or documentation for software culture. A GitHub account alone is not evidence of testing, CI, packaging or mentorship.
3. Use funder records for grants and programme memberships; never infer funding from publication volume.
4. Use alumni pages or opt-in outcome records for mentoring/outcomes. Do not infer mentoring quality from fame, group size or citations.
5. Recheck every contact detail, position, opening and immigration statement on the day it is used. This repository records research intelligence, not a live directory.
